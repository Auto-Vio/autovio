## Что такое AutoVio?

Большинство инструментов ИИ выполняют лишь один шаг создания видео. AutoVio берёт на себя весь процесс целиком.

Вы описываете, что хотите — продукт, идею, историю. AutoVio пишет сценарий сцена за сценой, генерирует изображение для каждой сцены, анимирует эти изображения в видеоклипы и собирает всё вместе в редакторе временной шкалы. Вы экспортируете готовый MP4.

Весь конвейер работает на вашей собственной инфраструктуре. Вы используете свои API-ключи. Результат принадлежит вам.

```
Текстовый запрос  →  Сценарий (LLM)  →  Изображения (Gemini / DALL-E)  →  Видеоклипы (Veo / Runway)  →  Монтаж  →  Экспорт
```

## Конвейер

AutoVio разбивает производство видео на пять этапов, которые отражают работу человеческой команды:

| Этап | Что происходит |
|------|----------------|
| **1 · Инициализация** | Задайте тему, аудиторию, разрешение, режим и дополнительные справочные материалы |
| **2 · Анализ** | Загрузите референсное видео — ИИ-зрение извлекает стиль, тон, темп и цвета |
| **3 · Сценарий** | LLM пишет сценарий сцена за сценой с подсказками для изображений, видео и переходами |
| **4 · Генерация** | Каждая сцена получает сгенерированное ИИ изображение, которое затем анимируется в видеоклип |
| **5 · Редактор** | Расставьте клипы на временной шкале, добавьте текстовые/графические оверлеи, задайте переходы, смикшируйте звук, экспортируйте |

Два режима генерации:
- **Перенос стиля** — Воспроизведение визуального стиля существующего видео на новый контент
- **Ремикс контента** — Создание с нуля с использованием руководства по стилю проекта и ваших запросов

## Ключевые возможности

- **Полный сквозной конвейер** — одна система от идеи до экспортированного MP4
- **Мультипровайдерный ИИ** — комбинируйте LLM, модели изображений и видеомодели для каждого проекта
- **Анализ референсного видео** — ИИ-зрение декодирует стиль, темп и композицию из любого видео
- **Руководства по стилю проекта** — зафиксируйте голос бренда, цветовую палитру, стиль камеры и тон один раз; применяйте ко всем видео
- **Библиотека ресурсов** — загружайте фотографии продуктов, логотипы или скриншоты; используйте их напрямую в видео или в качестве стилевых ссылок
- **Редактор временной шкалы** — текстовые оверлеи, графические оверлеи, переходы, микширование аудио, точная обрезка по кадрам
- **Система шаблонов** — сохраняйте композиции оверлеев как многократно используемые шаблоны
- **Управление разрешением** — Портрет 9:16, Альбомная 16:9 или Квадрат 1:1; каждый провайдер автоматически получает правильный формат
- **REST API + OpenAPI** — каждая функция доступна программно
- **MCP server** — используйте AutoVio из Claude Code, Cursor, Claude Desktop или любого MCP-клиента
- **Самостоятельный хостинг** — работает на вашем компьютере или сервере; данные не покидают систему без ваших API-ключей

## Провайдеры ИИ

AutoVio не зависит от конкретного провайдера. Настраивайте разных провайдеров для каждой роли:

| Роль | Поддерживаемые провайдеры |
|------|--------------------------|
| **LLM (сценарий)** | Google Gemini, OpenAI, Anthropic Claude |
| **Зрение (анализ)** | Google Gemini |
| **Генерация изображений** | Google Gemini Image, OpenAI DALL-E 3 |
| **Генерация видео** | Google Veo, Runway Gen-3 |

Новые провайдеры можно добавить, реализовав интерфейс `IImageProvider` или `IVideoProvider`.

## Сценарии использования

### Разработчики и ИИ-ассистенты программирования

AutoVio имеет полноценный MCP server. Ваш ИИ-ассистент программирования может генерировать демонстрационные видео продуктов, не покидая редактор:

- **Claude Code** — запустите `autovio_works_create` после выпуска функции
- **Cursor** — генерируйте обучающие видео для изменений кода прямо в редакторе
- **Claude Desktop** — опишите видео в разговоре, и оно будет создано автоматически

### Автоматизированные рабочие процессы

REST API подключается к любой платформе автоматизации:

- **n8n / Make / Zapier** — запускайте генерацию видео из вебхуков, событий CRM или расписаний
- **CI/CD конвейеры** — автоматически генерируйте видео с анонсами релизов при каждом развёртывании
- **Контент-календари** — массово создавайте видео для социальных сетей по контент-расписанию

### Команды по продукту и маркетингу

- Превращайте технические спецификации в демонстрационные видео продуктов
- Генерируйте локализованные варианты видео из единого сценария
- Создавайте обучающие видео из документации
- Поддерживайте единство бренда во всех видео с помощью руководств по стилю

### Исследователи и разработчики

- Экспериментируйте с новыми провайдерами ИИ-видео без перестройки инфраструктуры
- Используйте REST API как бэкенд для вашего собственного видеопродукта
- Расширяйте конвейер с помощью кастомных провайдеров, промптов или форматов экспорта

## Быстрый старт

### Требования

- **[Bun](https://bun.sh/)** >= 1.0 (или Node.js >= 18)
- **[MongoDB](https://www.mongodb.com/)** — локальный или [Atlas](https://www.mongodb.com/cloud/atlas)
- **FFmpeg** — для экспорта видео (`brew install ffmpeg` / `apt install ffmpeg`)
- Как минимум один API-ключ провайдера ИИ (Google Gemini бесплатен для начала)

### 1. Клонировать и установить

```bash
git clone https://github.com/Auto-Vio/autovio.git
cd autovio
bun install
```

### 2. Настроить

```bash
cp .env.example .env
# Откройте .env и задайте MONGODB_URI и JWT_SECRET
```

| Переменная | Обязательна | Описание |
|------------|-------------|----------|
| `MONGODB_URI` | Да | Строка подключения MongoDB |
| `JWT_SECRET` | Да | Секрет для JWT-токенов |
| `PORT` | Нет | Порт бэкенда (по умолчанию: 3001) |

### 3. Запустить

```bash
bun run dev
```

- Фронтенд: `http://localhost:5173`
- Бэкенд API: `http://localhost:3001`
- Документация OpenAPI: `http://localhost:3001/api/docs`

## MCP Server

Репозиторий [`AutoVio-MCP`](https://github.com/Auto-Vio/autovio-mcp) поставляется с полноценным MCP server, содержащим более 25 инструментов, охватывающих весь AutoVio API. Подключите его к Claude Desktop, Cursor или любому MCP-совместимому клиенту и генерируйте видео через разговор.

```json
{
  "mcpServers": {
    "autovio": {
      "command": "node",
      "args": [
        "/path/to/AutoVio-MCP/dist/index.js",
        "--autovio-base-url", "http://localhost:3001",
        "--autovio-api-token", "YOUR_TOKEN",
        "--llm-model", "gemini-2.5-flash",
        "--llm-api-key", "YOUR_KEY",
        "--image-model", "gemini-2.5-flash-image",
        "--image-api-key", "YOUR_KEY",
        "--video-model", "veo-3.0-generate-001",
        "--video-api-key", "YOUR_KEY"
      ]
    }
  }
}
```

См. [документацию MCP](https://auto-vio.github.io/autovio-docs/mcp/overview/) для полного руководства по настройке и справочника по инструментам.

## Участие в разработке

AutoVio находится на раннем этапе и активно развивается. Вклад приветствуется в любой форме:

- **Отчёты об ошибках** — откройте задачу с шагами для воспроизведения
- **Новые провайдеры ИИ** — реализуйте `IImageProvider` или `IVideoProvider` и откройте PR
- **Улучшения UI** — фронтенд использует React + TailwindCSS + Zustand
- **Документация** — сайт с документацией находится в [AutoVio-Docs](https://github.com/Auto-Vio/autovio-docs)
- **Идеи и обратная связь** — откройте обсуждение или задачу

Для начала ознакомьтесь с [документацией](https://auto-vio.github.io/autovio-docs/) и изучите кодовую базу. Интерфейсы провайдеров в `packages/backend/src/providers/interfaces.ts` — хорошая точка входа для добавления новых ИИ-интеграций.
